# Sber2Excel

Конвертер PDF-выписок дебетовых карт Сбербанка в CSV и Excel.

**Для всех остальных выписок используйте приложение от Ev2geny** [`Sberbank2Excel`](https://github.com/Ev2geny/Sberbank2Excel/)

Это **ядро** приложения — библиотека на Avalonia без точки входа. Содержит парсеры PDF, view-модели, представления (`MainView`, `MainWindow`) и сервисы экспорта. Запускается через одну из «голов»:

- [`Sber2Excel.Desktop`](https://github.com/glebasos/Sber2Excel.Desktop) — десктопное приложение (Windows / Linux / macOS).
- [`Sber2Excel.Browser`](https://github.com/glebasos/Sber2Excel.Browser) — WebAssembly, работает прямо в браузере.

## Что делает

Загружает PDF-выписку, скачанную из СберБанк Онлайн (раздел «Выписки и справки»), разбирает все операции и позволяет сохранить результат в:

- **CSV** — UTF-8 BOM, разделитель `;`, удобно для Excel
- **XLSX** — два листа: «Информация» (реквизиты счёта, итоги) и «Операции» (таблица транзакций с цветовым выделением поступлений и списаний)

## Возможности UI

- Шапка с владельцем счёта, картой, периодом, остатками, пополнениями и списаниями
- `TreeDataGrid` со всеми операциями, сортировка по любой колонке
- Кнопка «Сбросить сортировку»
- Фильтры: по категории (выпадающий список) и по описанию (поиск-подстрока)
- Строка статуса с количеством операций и сообщениями об ошибках

## Структура проекта

```
Sber2Excel/                       ← это ядро (библиотека)
  App.axaml(.cs)                  ← Avalonia App, поддерживает оба lifetime
  Views/MainView.axaml            ← общий UserControl для обеих «голов»
  Views/MainWindow.axaml          ← окно-обёртка для десктопа
  ViewModels/MainWindowViewModel  ← CommunityToolkit.Mvvm
  Services/Parsing/               ← PdfParserFactory + парсеры банков (PdfPig)
  Services/ExportService.cs       ← CSV + XLSX (ClosedXML)
  Models/                         ← StatementInfo, Transaction
  Converters/                     ← BoolToColorConverter
```

## Поддерживаемый формат

Выписки по **дебетовым картам Сбербанка**, экспортированные из СберБанк Онлайн в PDF.
Многостраничный документ с таблицей операций:

| Дата операции (МСК) | Категория | Сумма в валюте счёта | Остаток |
|---|---|---|---|
| Дата обработки + код авт. | Описание операции | — | — |

Протестировано на выписках за 2025 год (A4, до 20 страниц).

## Колонки в экспорте

| Колонка | Описание |
|---|---|
| Дата операции | Дата и время по МСК |
| Дата обработки | Дата проведения по счёту |
| Код авторизации | 5–6-значный код банка |
| Категория | Тип операции (Перевод СБП, Прочие операции и т.д.) |
| Описание | Детали: получатель/отправитель, источник |
| Сумма | Положительная — поступление, отрицательная — списание |
| Остаток | Остаток на счёте после операции |

## Добавление парсера для нового банка

1. Создайте класс в `Services/Parsing/`, наследующий `PdfParserBase`.
2. Реализуйте `CanParse(firstPageText)` с уникальным признаком документа.
3. Реализуйте обе перегрузки `Parse(string filePath)` и `Parse(byte[] pdfBytes)` — браузерная «голова» работает только с байтами.
4. Зарегистрируйте парсер в `PdfParserFactory`.

## Ограничения

- Только рублёвый счёт (операции в иностранной валюте отображаются корректно, но валюта операции не экспортируется).
- Только дебетовые карты Сбербанка. Кредитки и другие банки не поддерживаются.

## Замечания о тримминге (важно для браузерной «головы»)

- XAML использует компилируемые биндинги (`x:CompileBindings="True"`).
- Биндинги, построенные в коде через `new Binding(nameof(...))`, — **рефлексивные**, тримминг вырезает целевые свойства. Поэтому ячейки колонок «Сумма» и «Остаток» (`TemplateColumn`) читают свойства `Transaction` напрямую в обработчике `DataContextChanged`, без `Binding`.
